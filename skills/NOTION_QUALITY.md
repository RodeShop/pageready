# NOTION_QUALITY — Чеклист качества продукта

_MiMo проверяет каждый продукт перед переносом в ready/_

---

## Notion Template — обязательно

### Структура (без этого = брак)

- [ ] **Минимум 3 базы данных** — связанные через Relations (Clients → Projects → Tasks или эквивалент ниши)
- [ ] **Минимум 1 formula** — Overdue, Progress%, Balance (см. `NOTION_BLOCKS.md`)
- [ ] **Минимум 1 rollup** — Total Budget, Task Count, Hours (см. `NOTION_BLOCKS.md`)
- [ ] **Минимум 3 вьюхи** на главной базе (Table + Board + Calendar / Timeline / Gallery)
- [ ] **Примеры данных** — не пустой шаблон. Минимум 6 примеров в главной базе, 4+ в дочерних
- [ ] **Страница Welcome / How to Use** — первое что видит покупатель
- [ ] **Все поля именованы** — не "Property 1", а говорящие названия

> **Legacy (до TASK-012):** 2 базы без formula/rollup = оценка C, не публикуем новые. Продукт #1 на Gumroad — исключение, ждёт upgrade.

### Нишевость (без этого не продаётся)

- [ ] **Конкретная аудитория** — название содержит кто это (freelancers, creators, students)
- [ ] **Конкретная боль** — описание объясняет какую проблему решает
- [ ] **Не generic** — "Project Tracker" плохо, "Freelance Client Tracker for Designers" хорошо

### Качество контента

- [ ] **Примеры реалистичны** — не "Example 1", "Task A", а реальные названия проектов/задач
- [ ] **Эмодзи на страницах** — визуально структурирует
- [ ] **Формулы/фильтры настроены** — если есть поля дат/статусов — есть фильтры по ним
- [ ] **Инструкция понятна** — покупатель без объяснений разберётся за 5 минут

### Листинг (без этого нет продаж)

- [ ] **Заголовок до 60 символов** — содержит нишу и аудиторию
- [ ] **Описание 200–300 слов** — benefits first, не features
- [ ] **10 тегов** — нишевые + общие
- [ ] **Цена обоснована** — $29 для template с 2+ базами, $19 для простого
- [ ] **Thumbnail concept** — описано что должно быть на обложке

---

## AI Prompt Bundle — обязательно

### Объём и качество

- [ ] **Минимум 50 промптов** — разбиты по подкатегориям (5–10 групп)
- [ ] **Каждый промпт работает** — конкретный, с переменными в [скобках]
- [ ] **System prompt** — один главный system prompt для настройки ассистента
- [ ] **Usage guide** — как использовать, в каких инструментах (ChatGPT, Claude, etc.)
- [ ] **Примеры результатов** — 3–5 примеров "prompt → output"

### Нишевость

- [ ] **Одна конкретная профессия/задача** — не "маркетинг вообще", а "email маркетинг для e-commerce"
- [ ] **Язык аудитории** — промпты написаны в контексте ниши
- [ ] **Актуально** — нет устаревших инструментов или практик

---

## Оценка продукта

После проверки поставь оценку:

- **A** — все пункты выполнены, продукт готов
- **B** — 1–2 незначительных пункта, можно публиковать
- **C** — есть серьёзные пробелы, нужна доработка → создать тикет

Только A и B идут в `products/ready/`.
